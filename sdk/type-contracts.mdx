The SOIL SDK exports TypeScript types for every output object in the pipeline. Use these types to build type-safe integrations — they describe the exact shape of every object produced by a builder and consumed by a validator. Import from `functions/src/soil/models` for types and from `functions/src/soil/constants` for schema version strings and sector family identifiers.

## Import Paths

```typescript
// Types
import type {
  SoilCompletionScenarioRun,
  SoilCompletionSectorPackConfig,
  SoilCompletionExecutiveReadinessAssessment,
  SoilCompletionExportReadyObjectPackage,
} from 'functions/src/soil/models';

// Constants
import {
  SOIL_COMPLETION_SCHEMA_VERSIONS,
  SOIL_COMPLETION_SUPPORTED_SECTOR_FAMILIES,
} from 'functions/src/soil/constants';
```

You can also import everything through the top-level barrel:

```typescript
import type { SoilCompletionScenarioRun } from 'functions/src/soil/models';
import { SOIL_COMPLETION_SCHEMA_VERSIONS } from 'functions/src/soil/constants';
// or from the root barrel:
import { SOIL_COMPLETION_SCHEMA_VERSIONS } from 'functions/src/soil';
```

## Schema Version Constants

`SOIL_COMPLETION_SCHEMA_VERSIONS` is a constant object exported from `functions/src/soil/constants`. Use these string values to verify schema version fields on any SOIL object.

```typescript
import { SOIL_COMPLETION_SCHEMA_VERSIONS } from 'functions/src/soil/constants';

// Verify a received object before processing
if (sectorPackConfig.schemaVersion !== SOIL_COMPLETION_SCHEMA_VERSIONS.sector_pack_config_v2) {
  throw new Error('Unexpected SectorPackConfig schema version');
}
```

**Available Keys**

| Key | Value String | Applies To |
|---|---|---|
| `sector_pack_config_v2` | `"sector_pack_config_v2"` | `SoilCompletionSectorPackConfig` |
| `scenario_run_v2` | `"scenario_run_v2"` | `SoilCompletionScenarioRun` |
| `workplace_condition_input_v1` | `"workplace_condition_input_v1"` | `SoilCompletionWorkplaceConditionInput` |
| `workplace_compliance_output_v1` | `"workplace_compliance_output_v1"` | `SoilCompletionWorkplaceComplianceOutput` |
| `readiness_horizon_projection_v1` | `"readiness_horizon_projection_v1"` | `SoilCompletionReadinessHorizonProjection` |
| `projected_vs_observed_v1` | `"projected_vs_observed_v1"` | `SoilCompletionProjectedVsObserved` |
| `executive_readiness_assessment_v1` | `"executive_readiness_assessment_v1"` | `SoilCompletionExecutiveReadinessAssessment` |
| `map_consumable_output_v1` | `"map_consumable_output_v1"` | `SoilCompletionMapConsumableOutput` |
| `human_environment_fusion_v1` | `"human_environment_fusion_v1"` | `SoilCompletionHumanEnvironmentFusion` |
| `ocm10_handoff_v1` | `"ocm10_handoff_v1"` | `SoilCompletionOcm10Handoff` |
| `reality_feedback_hook_v1` | `"reality_feedback_hook_v1"` | `SoilCompletionRealityFeedbackHook` |
| `benchmark_ready_output_v1` | `"benchmark_ready_output_v1"` | `SoilCompletionBenchmarkReadyOutput` |
| `observability_trace_hook_v1` | `"observability_trace_hook_v1"` | `SoilCompletionObservabilityTraceHook` |
| `protected_use_status_v1` | `"protected_use_status_v1"` | `SoilCompletionProtectedUseStatus` |

<Note>
  Schema versions follow the pattern `{object_name}_v{N}`. Sector pack config and scenario run are at `v2` — all other contracts are currently at `v1`. When BEL publishes a schema update, the version key value will change and validators will enforce the new version.
</Note>

## Supported Sector Families

`SOIL_COMPLETION_SUPPORTED_SECTOR_FAMILIES` lists the active sector pack IDs recognized by the SOIL backend.

```typescript
import { SOIL_COMPLETION_SUPPORTED_SECTOR_FAMILIES } from 'functions/src/soil/constants';

// Use in request validation
if (!SOIL_COMPLETION_SUPPORTED_SECTOR_FAMILIES.active.includes(request.sectorPackId)) {
  // Reject — not an active sector family
}
```

**Active Sector Pack IDs**

| Sector Pack ID | Builder Function | Status |
|---|---|---|
| `hbx_workplace_readiness_compliance` | `buildDefaultSoilCompletionSectorPackConfig()` | Active |
| `capital_public_finance` | `buildCapitalPublicFinancePackConfig()` | Active |
| `revenue_commercial` | `buildRevenueCommercialPackConfig()` | Active |

**Requires Review**

| Sector Pack ID | Status | Notes |
|---|---|---|
| `aviation` / HBX Aero | `requires_review` | No active Aviation/HBX Aero code path is present in the current SOIL backend. The aviation sector family is explicitly marked `requires_review` and cannot be used in active scenario runs until a BEL-provided backend path is confirmed. |

<Warning>
  Do not pass `aviation` or any HBX Aero sector pack ID in a live scenario run request. `validateSoilCompletionScenarioRunRequest` will reject it with a `requires_review` status code. This gap is intentionally documented and will be resolved in a future milestone.
</Warning>

## Approval Posture Constants

All SOIL objects that carry an `approvalStatus` field must use a value from the governed posture vocabulary. Prohibited language — `certified`, `authorized`, `compliance_pass`, or unsupported external-validation claims — will cause validation to fail.

```typescript
import { SOIL_COMPLETION_APPROVAL_POSTURE } from 'functions/src/soil/constants';

// Example: check that an object is in a review-ready state
if (assessment.approvalStatus === SOIL_COMPLETION_APPROVAL_POSTURE.ready_for_review) {
  // Safe to route for human review
}
```

**Allowed Posture Values**

| Constant Key | Value | Meaning |
|---|---|---|
| `not_assessed` | `"not_assessed"` | No assessment has been run against this object yet. |
| `planned` | `"planned"` | Assessment is planned but has not started. |
| `in_progress` | `"in_progress"` | Assessment is currently underway. |
| `ready_for_review` | `"ready_for_review"` | Object is complete and ready for human review. **Default posture for all SOIL builder outputs.** |
| `verified_internal` | `"verified_internal"` | Object has passed internal verification. Does not constitute external validation or certification. |
| `external_validation_required` | `"external_validation_required"` | External validation is required before this object can be used for its intended purpose. |
| `requires_review` | `"requires_review"` | Object or sector family requires additional review before use (for example, the aviation sector family). |

<Note>
  All objects produced by SOIL builders default to `approvalStatus: ready_for_review`. BEL final approval is not claimed at this posture level — `ready_for_review` means the object is ready for a human reviewer to evaluate, not that it has been approved or certified.
</Note>
