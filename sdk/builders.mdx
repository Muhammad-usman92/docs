Builder functions in BEL SOIL construct validated, governance-compliant output objects for each schema type. Every builder enforces the required safety posture — `decisionSupportOnly: true`, `humanReviewRequired: true`, and `approvalStatus: READY_FOR_REVIEW` — so the objects they return are ready to pass to validators, downstream services, or export pipelines without further configuration.

## Import Path

Import any builder directly from the package barrel:

```typescript
import {
  buildDefaultSoilCompletionSectorPackConfig,
  createSoilCompletionScenarioRun,
  buildExecutiveReadinessAssessment,
  buildExportReadyObjectPackage,
  createSoilCompletionPhase1SamplePackage,
  // ...etc
} from '@bel/soil';
```

## Sector Pack Builders

Sector pack builders return a `SoilCompletionSectorPackConfig` object that governs which sector family is active, what guardrails apply, and what the approval posture is. Pass the result directly to `createSoilCompletionScenarioRun` or validate it first with `validateSoilCompletionSectorPackConfig`.

### `buildDefaultSoilCompletionSectorPackConfig()`

Builds the default `SectorPackConfig v2` — the HBX Workplace Readiness & Compliance pack. Use this as your starting point for any workplace scenario.

**Signature**

```typescript
function buildDefaultSoilCompletionSectorPackConfig(): SoilCompletionSectorPackConfig
```

**Returns** `SoilCompletionSectorPackConfig`

**Usage**

```typescript
import { buildDefaultSoilCompletionSectorPackConfig } from '@bel/soil';

const config = buildDefaultSoilCompletionSectorPackConfig();
// config.sectorPackId === 'hbx_workplace_readiness_compliance'
// config.sectorPackVersion === 'v2'
// config.decisionSupportOnly === true
// config.humanReviewRequired === true
// config.approvalStatus === 'READY_FOR_REVIEW'
```

<Note>
  All sector pack configs produced by SOIL builders set `decisionSupportOnly: true` and `humanReviewRequired: true` unconditionally. These fields cannot be overridden downstream.
</Note>

## Scenario Run Builders

Scenario run builders assemble the core `SoilCompletionScenarioRun` object from structured inputs. The scenario run is the primary container that carries conditions, events, state transitions, and governance metadata through the rest of the pipeline.

### `createSoilCompletionScenarioRun(sectorPackConfig, conditions, events, options)`

Assembles a full scenario run object from discrete inputs. Use this when you are constructing a scenario programmatically from individual components.

**Signature**

```typescript
function createSoilCompletionScenarioRun(
  sectorPackConfig: SoilCompletionSectorPackConfig,
  conditions: SoilCompletionWorkplaceConditionInput[],
  events: object[],
  options: {
    scenarioId: string;
    soilSessionId: string;
    tenantId: string;
    organizationId: string;
    programId: string;
    correlationId: string;
    auditReference: string;
    isSynthetic?: boolean;
    assumptions?: string[];
  }
): SoilCompletionScenarioRun
```

**Returns** `SoilCompletionScenarioRun`

**Usage**

```typescript
import {
  buildDefaultSoilCompletionSectorPackConfig,
  createSoilCompletionScenarioRun,
} from '@bel/soil';

const config = buildDefaultSoilCompletionSectorPackConfig();

const scenarioRun = createSoilCompletionScenarioRun(
  config,
  [/* conditions */],
  [/* events */],
  {
    scenarioId: 'scenario_001',
    soilSessionId: 'session_001',
    tenantId: 'tenant_demo',
    organizationId: 'org_demo',
    programId: 'program_demo',
    correlationId: 'corr_001',
    auditReference: 'audit_001',
    isSynthetic: true,
    assumptions: ['All condition data is source-authorized.'],
  }
);
```

### `buildSoilCompletionSafeFailureResponse(rejectionReasons, auditReference, correlationId)`

Produces a governed safe failure response when a scenario run or input is rejected. The response includes a `safeFallbackOutput` with all prohibited-use declarations and a `sanitizedAuditEvent`. No unsafe output is generated from rejected input.

**Signature**

```typescript
function buildSoilCompletionSafeFailureResponse(
  rejectionReasons: string[],
  auditReference: string,
  correlationId: string
): {
  ok: false;
  outputStatus: 'rejected';
  rejectionReasons: string[];
  safeFallbackOutput: SoilCompletionSafeFailureResponse;
  sanitizedAuditEvent: SoilCompletionSanitizedAuditEvent;
}
```

**Usage**

```typescript
import { buildSoilCompletionSafeFailureResponse } from '@bel/soil';

const failure = buildSoilCompletionSafeFailureResponse(
  ['unknown_condition:unknown_condition'],
  'audit_failure',
  'corr_failure'
);

// failure.ok === false
// failure.outputStatus === 'rejected'
// failure.safeFallbackOutput.decisionSupportOnly === true
// failure.safeFallbackOutput.humanReviewRequired === true
```

<Warning>
  Never pass a rejected safe failure response to downstream decisioning systems as if it were a valid scenario output. The `ok: false` field signals that the pipeline must halt and route the event to human review.
</Warning>

## Compliance and Fusion Builders

### `evaluateWorkplaceCompliance(inputs, options)`

Runs workplace suppression logic and builds a `SoilCompletionWorkplaceComplianceOutput`. Groups below the small-group threshold are suppressed and rolled up to an approved higher level. The output includes required workplace disclaimers and a multi-signal convergence flag.

**Signature**

```typescript
function evaluateWorkplaceCompliance(
  inputs: SoilCompletionWorkplaceConditionInput[],
  options: {
    outputId: string;
    correlationId: string;
    auditReference: string;
    isSynthetic?: boolean;
  }
): SoilCompletionWorkplaceComplianceOutput
```

**Returns** `SoilCompletionWorkplaceComplianceOutput`

**Usage**

```typescript
import { evaluateWorkplaceCompliance } from '@bel/soil';

const complianceOutput = evaluateWorkplaceCompliance(
  [/* workplace condition inputs */],
  {
    outputId: 'workplace_output_001',
    correlationId: 'corr_wp_001',
    auditReference: 'audit_wp_001',
    isSynthetic: true,
  }

```

<Note>
  Workplace compliance output explicitly blocks legal determinations, allegation truth scoring, credibility scoring, and automated employment actions. The `humanReviewRequired` field is always `true` on every returned object.
</Note>

  correlationId: string;
  auditReference: string;
  isSynthetic?: boolean;

import { buildRealityFeedbackHook } from '@bel/soil';

  `automaticRecalibration` is always `false`. SOIL does not support autonomous recalibration or automatic production model changes. Any attempt to set this field to `true` will fail validation.

### `createSoilCompletionPhase1SamplePackage()`

Generates the complete Phase 1 sample package in a single call — all output objects, fully populated with synthetic data, ready to inspect or test against. Use this for integration testing, acceptance verification, or exploring the full SOIL object graph.

**Signature**

```typescript
function createSoilCompletionPhase1SamplePackage(): SoilCompletionPhase1SamplePackage
```

**Returns** `SoilCompletionPhase1SamplePackage`

**Usage**

```typescript
import { createSoilCompletionPhase1SamplePackage } from '@bel/soil';

const sample = createSoilCompletionPhase1SamplePackage();

// Inspect individual objects:
console.log(sample.sectorPackConfig);
console.log(sample.executiveReadinessAssessment);
console.log(sample.exportReadyObjectPackage);
```

**Returned Object Properties**

| Property | Type | Description |
|---|---|---|
| `sectorPackConfig` | `SoilCompletionSectorPackConfig` | The default HBX Workplace Readiness & Compliance sector pack config (v2). |
| `scenarioRun` | `SoilCompletionScenarioRun` | A fully assembled scenario run with synthetic conditions and events. |
| `workplaceOutput` | `SoilCompletionWorkplaceComplianceOutput` | Workplace compliance output with suppression and rollup applied. |
| `mapConsumableOutput` | `SoilCompletionMapConsumableOutput` | Map-consumable data contract (no GIS rendering). |
| `humanEnvironmentFusion` | `SoilCompletionHumanEnvironmentFusion` | Human-environment fusion object combining workplace and EOS signals. |
| `ocm10Handoff` | `SoilCompletionOcm10Handoff` | OCM-10 handoff object referencing the fusion ID. |
| `realityFeedbackHook` | `SoilCompletionRealityFeedbackHook` | Reality feedback hook with `automaticRecalibration: false`. |
| `benchmarkReadyOutput` | `SoilCompletionBenchmarkReadyOutput` | Benchmark metadata with `activeBenchmarking: false`. |
| `readinessHorizonProjection` | `SoilCompletionReadinessHorizonProjection` | Four-case readiness horizon: base, upside, downside, and stress. |
| `projectedVsObserved` | `SoilCompletionProjectedVsObserved` | Projected vs. observed comparison with automatic model change disabled. |
| `executiveReadinessAssessment` | `SoilCompletionExecutiveReadinessAssessment` | Executive assessment with Evidence, Assumption, and Confidence ledgers. |
| `observabilityTrace` | `SoilCompletionObservabilityTraceHook` | Observability trace linking all object IDs to the audit reference. |
| `protectedUseStatus` | `SoilCompletionProtectedUseStatus` | Protected-use governance marker for the pipeline run. |

<Tip>
  Run `createSoilCompletionPhase1SamplePackage()` in your acceptance tests to verify that the entire object graph is structurally valid before passing individual objects to validators or downstream services. Every property on the returned package can be independently passed to its matching validator.
</Tip>
